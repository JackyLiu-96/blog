1. 全文搜索引擎

  • 全文搜索引擎是目前广泛应用的主流搜索引擎。它的工作原理是计算机索引程序通过扫描文章中的每一个词，对每一个词建立一个索引，指明该词在文章中出现的次数和位置，当用户查询时，检索程序就根据事先建立的索引进行查找，并将查找的结果反馈给用户的检索方式。这个过程类似于通过字典中的检索字表查字的过程。

  • 常用的全文搜索引擎

  • Lucene，目前最受欢迎的Java全文搜索框架。原因很简单，hibernate search、solr、elasticsearch都是基于lucene拓展出来的搜索引擎。

  • Hibernate Search是在apache Lucene的基础上建立的主要用于Hibernate的持久化模型的全文检索工具。

  • ElasticSearch也使用Java开发并使用Lucene作为其核心来实现所有索引和搜索的功能，但是它的目的是通过简单的RESTful API来隐藏Lucene的复杂性，从而让全文搜索变得简单。

  • Solr它是一种开放源码的、基于 Lucene Java 的搜索服务器，易于加入到 Web 应用程序中。提供了层面搜索(就是统计)、命中醒目显示并且支持多种输出格式（包括XML/XSLT 和JSON等格式）。

  • 我们重点学习es

1.1. ElasticSearch简介

1.1.1. 什么是ElasticSearch

  • Elasticsearch是一个实时的分布式搜索和分析引擎。它可以帮助你用前所未有的速度去处理大规模数据。

  • ElasticSearch是一个基于Lucene的搜索服务器。它提供了一个分布式多用户能力的全文搜索引擎，基RESTful

    web接口。Elasticsearch是用Java开发的，并作为Apache许可条款下的开放源码发布，是当前流行的企业级

    搜索引擎。设计用于云计算中，能够达到实时搜索，稳定，可靠，快速，安装使用方便。

1.1.2. ElasticSearch特点

  1. 可以作为一个大型分布式集群（数百台服务器）技术，处理PB级数据，服务大公司；也可以运行在单机上
  2. 将全文检索、数据分析以及分布式技术，合并在了一起，才形成了独一无二的ES；
  3. 开箱即用的，部署简单
  4. 全文检索，同义词处理，相关度排名，复杂数据分析，海量数据的近实时处理

逻辑结构部分是一个倒排索引表：

1.7. elasticsearch与MySQL数据同步

  • ES在项目使用，其它ES就是搜索的数据库，它能实现CRUD，在搜索上能实现全文检索
  • 原始的数据其实都在MySQL数据库，如何保存到es数据库
    • 手动保存
      • mysql增加数据，手动同步es
      • mysql修改数据，手动同步es
      • mysql删除数据，手动同步es
    • logstash自动保存
      • mysql增加数据，自动同步es
      • mysql修改数据，自动同步es
      • mysql删除数据， 手动同步es
